# 🔗 MyLink (마이링크)

> **단 하나의 링크로 당신의 모든 활동을 연결하세요.**
>
> **MyLink**는 깃허브, 블로그, 포트폴리오, SNS 채널 등 흩어져 있는 나만의 디지털 발자국을 미니멀하고 직관적인 단 하나의 프로필 페이지로 통합하고 분석할 수 있는 **Linktree 클론 서비스**입니다.

---

## ✨ 핵심 기능 (Key Features)

### 1. 📱 가상 미리보기 및 인터랙티브 랜딩
* **실시간 가상 모바일 Sandbox**: 메인 페이지에 진입한 비로그인 사용자가 직접 링크를 클릭해보며 클릭 카운트의 증가와 이모지 폭죽 이펙트를 체험해 볼 수 있는 인터랙티브 놀이터를 제공합니다.
* **둥실둥실 소셜 갤러리**: 주요 개발자 플랫폼 채널 아이콘들이 마우스 호버 시 떠오르는 입체 모션 그래픽을 탑재하여 즐거운 사용자 경험을 제공합니다.

### 2. 🔗 직관적인 링크 관리 (CRUD)
* **Favicon 자동 파싱**: 입력한 URL의 도메인을 기반으로 구글 Favicon API를 통해 고해상도 파비콘을 실시간 추출하여 매핑합니다.
* **인라인 빠른 편집**: 복잡한 페이지 이동 없이 링크 목록 카드 내에서 직접 제목과 주소를 빠르게 수정할 수 있습니다.
* **TanStack Query (React Query) 최적화**: 낙관적 업데이트(Optimistic Update)를 적용하여 링크의 추가, 수정, 삭제 요청이 네트워크 지연 없이 즉각적으로 화면에 반영됩니다.

### 3. 📊 실시간 클릭 통계 분석 (`/stats`)
* **누적 클릭수 강조**: 등록한 모든 소셜 링크의 총 누적 조회수를 프리미엄 보라색 포인트 컬러로 크게 시각화합니다.
* **shadcn/ui 차트 분석**: 가장 많은 트래픽을 유도한 **인기 링크 Top 5**를 가로형 막대 차트(Horizontal Bar Chart)로 직관적으로 매핑합니다.
* **최대 클릭 대비 도달율 진행 바**: 가장 성과가 높은 1위 링크의 클릭수를 기준으로 각 링크가 몇 %의 상대적 성과를 달성했는지 보라색 진행 바(Progress Bar)를 통해 한눈에 파악합니다.
* **보안 리다이렉트**: 비로그인 사용자가 직접 접근을 시도할 시 `router.replace`를 사용하여 즉각적이고 안전하게 강제 퇴거시킵니다.

### 4. 👤 인스타그램 스타일 프로필 관리
* **프로필 설정**: 이름, 한 줄 소개(최대 80자), 그리고 계정 사진을 한 곳에서 수정하는 반응형 모달을 탑재했습니다.
* **고유 Username 도메인**: 나만을 위한 유니크 주소(`mylink-bice.vercel.app/{username}`)를 가질 수 있으며, 주소 복사 및 새 창 미리보기 편의 기능을 상단 헤더에 완비했습니다.

### 5. 🎨 프리미엄 비주얼 디자인
* **보라색 `#5B5FC7` 아이덴티티**: 서비스 곳곳에 은은하게 흐르는 그라데이션 광원 효과와 부드러운 카드 그림자 테두리를 배치했습니다.
* **동적 OpenGraph 이미지**: `next/og` 엔진 및 Edge Runtime을 활용하여 카카오톡, 슬랙 등에 내 프로필 공유 시 `1200x630` 해상도의 `[username]의 마이링크` 맞춤형 썸네일 카드가 실시간 생성됩니다.
* **고해상도 3D 파비콘**: 브라우저 탭에 표시되는 대표 파비콘을 고품질 3D 링크 아이콘(`app/icon.png`)으로 전면 개편했습니다.

### 6. 🔐 견고한 구글 소셜 로그인
* **useRef 이중 클릭 방지**: 로그인 상태 변경 딜레이 및 React Strict Mode 더블 이펙트로 인한 이중 팝업 충돌을 동기식 플래그로 차단했습니다.
* **Redirect Fallback**: 일부 인앱 웹뷰나 광고 차단 확장 프로그램에 의해 팝업창(`signInWithPopup`)이 차단당할 경우, 즉시 화면 전환 로그인(`signInWithRedirect`)으로 자동 우회 처리하여 로그인 실패 가능성을 소멸시켰습니다.

## 🔒 Firestore 보안 규칙 (Security Rules)

데이터 무결성과 프라이버시 보호를 위해 정밀한 규칙 설정을 완수하였습니다.
* **users 컬렉션**: 본인의 프로필 문서만 작성 및 수정 가능 (`request.auth.uid == userId`)
* **links 서브컬렉션**: 
  - 생성 및 삭제는 오직 로그인 본인만 가능.
  - 외부/익명 사용자는 제목과 주소 등의 핵심 데이터를 훼손하지 않고 **오직 `clickCount` 필드를 `기존값 + 1`로 증가시키는 단일 필드 업데이트 요청만 허용**하도록 격리 조치.
