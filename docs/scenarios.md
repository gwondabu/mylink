# 마이링크 (MyLink) 사용자 시나리오

본 문서는 **마이링크(MyLink)** 서비스의 사용자 행동 흐름을 미니멀하게 정의한 사용자 시나리오 문서입니다. 모든 시나리오는 실제 서비스 사용 목적과 행동을 바탕으로 설계되었습니다.

---

## 1. 방문자 (Visitor) 시나리오

### 1.1 마이링크 페이지 방문 및 탐색
* **방문 목적 접속**
  * 사용자는 관심 있는 개발자나 크리에이터의 통합 링크 정보를 한눈에 확인하기 위해 **마이링크 프로필 페이지(`https://mylink.to/{username}`)에 접속한다.**
* **콘텐츠 이동**
  * 사용자는 프로필 주인이 소개하는 특정 사이트나 소셜 채널로 이동하기 위해 **화면에 나열된 링크 목록 중 원하는 링크 카드를 클릭한다.**
* **연결 방식 유지**
  * 사용자는 기존 마이링크 페이지를 종료하지 않고 계속 열어둔 채 다른 채널을 탐색하기 위해 **새 창(`_blank`)으로 열리는 외부 웹 페이지를 확인한다.**

---

## 2. 소유자 (Owner) 시나리오

### 2.1 회원가입 및 프로필 초기 설정
* **구글 로그인 인증**
  * 사용자는 비밀번호 입력 등의 번거로운 가입 절차 없이 마이링크 서비스를 즉시 시작하기 위해 **로그인 화면에서 구글 소셜 로그인 연동을 진행한다.**
* **고유 주소 설정**
  * 사용자는 나만의 개별 공유 페이지 주소를 선점하기 위해 **중복이 없는 고유 영문 `username`을 입력하여 회원 등록을 완료한다.**
* **프로필 기본 정보 설정**
  * 사용자는 방문자들에게 표시될 닉네임이나 브랜드 이름을 노출하기 위해 **대시보드에서 디스플레이 네임(displayName)을 입력한다.**
* **한 줄 소개 및 이미지 설정**
  * 사용자는 자신에 대한 짧은 소개와 얼굴(혹은 로고)을 시각적으로 나타내기 위해 **한 줄 소개글을 작성하고 프로필 이미지를 업로드한다.**

### 2.2 링크 관리 (추가 / 수정 / 삭제)
* **링크 추가**
  * 사용자는 방문자들에게 홍보하고 싶은 새로운 웹 포트폴리오나 소셜 주소를 추가하기 위해 **대시보드 내 링크 추가 영역에 링크 제목(Title)과 연결 주소(URL)를 입력한다.**
* **파비콘 노출 확인**
  * 사용자는 방문자가 링크의 성격을 시각적으로 더 쉽게 인지할 수 있도록 돕기 위해 **추가된 링크 옆에 자동으로 파싱되어 나타나는 파비콘 아이콘 이미지를 확인한다.**
* **링크 내용 수정**
  * 사용자는 기존에 등록된 링크의 제목 텍스트를 바꾸거나 변경된 URL 주소를 반영하기 위해 **대시보드의 해당 링크 카드 내 입력폼에서 정보를 직접 수정한다.**
* **링크 영구 삭제**
  * 사용자는 기간이 지나 더 이상 방문자들에게 노출할 필요가 없는 링크를 없애기 위해 **링크 카드 우측의 삭제 버튼을 클릭하여 링크를 완전히 제거한다.**

### 2.3 계정 로그아웃
* **세션 해제**
  * 사용자는 공용 디바이스 등에서 내 계정 정보와 대시보드 관리 화면을 안전하게 보호하기 위해 **로그아웃 버튼을 클릭하여 접속 세션을 종료한다.**
